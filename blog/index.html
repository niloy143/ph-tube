Question 1: Discuss the scope of var, let, and const

var

  • If a variable is declared using var, it would be scoped in the function where it's declared.
  • It can be accessed from anywhere inside that function, even before it's declaration, in that case the value of the variable would be undefined.

let

  • Variables declared using let would be scoped in the block where they are declared. A block can be indicated by curly braces (e.g. loops, if-else, switch-case, try-catch etc.)
  • It cannot be accessed before it's declared.

const

  • Variables declared using const would be scoped in the block where they are declared like 'let'.
  • It cannot be accessed before it's declared.

Question 2: Tell us the use cases of null and undefined

null

  • Null indicates the absence of a value in the variable.
  • It can be used to indicate that there should be a value that is currently missing.
  • A variable can be assigned null value if there is no value currently to store, but in future it would come.
  • If a variable loses a value, it can be assigned the null value to indicate that the value is currently unavailable.

undefined

  • When a variable isn't given any value yet, in JavaScript, it's given undefined by default.
  • Undefined is the default value of any variable that's not assigned any value.
  • Unlike null, undefined means the variable is blank, while null means the variable is currently absent.

Question 3: What do you mean by REST API?

API is the middleman between the client and the server, while REST API is like a set of operations or a system of the API's that arranges operations. In other words, REST API is a representation of the API which specifies the operations such as GET, POST, PUT etc.

  • REST stands for Representational State Transfer.
  • API stands for ApplicationProgrammingInterface.
  • REST API is used to arrange/specify the operations between the client and server.
  • When a client makes a certain request to the server, if it uses REST API methods, then the server can easily understand which type of operation the client is trying to make.

  • There are various kinds of operations in the REST method such as GET, POST, PUT, DELETE etc.

    • GET request means the client wants to read something from the server.
    • POST means the client wants to create/write something.
    • PUT means the client wants to update some existing values.
    • DELETE means the client wants to delete certain things.
